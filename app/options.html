Change Set Assistant

Adds a search bar and type filter to the Salesforce Change Set interface. Using the checkbox in the table header will only check the rows that are visible. Also adds a Show All link above the paging to show all items for the current item type.

Field Set API Names

When adding a field to a Field Set, this will add the API name next to the field name. There is a known Salesforce issue where certain related fields will all display as the same name (eg. BillingAddress or Name instead of BillingStreet and FirstName, respectively).

Setup Search

In the Setup area in Salesforce, if you try and use the quick search in the left sidebar before the placeholder text is loaded, your text will be appended to the placeholder text and your search will not find what you are looking for. This will prevent the placeholder text from being loaded by default.

Object/Field API Name

When creating a new object or field, this will automatically remove the underscores in the Object/Field Name field (API Name).

Field History Tracking Counter

Displays the total number of selected fields to track history for out of the total number Salesforce allows.

Setup Area Check All

When editing settings in the setup area, this will add a checkbox to the header row cells if there is a checkbox in that column. This will allow checking and unchecking all checkboxes in that column.

Add to Page Layout Uncheck All

When creating a new field, this setting will by default uncheck all checkboxes for adding the field to a page layout.

Allowed Number of Field History Fields

When setting the field history tracking fields, the specified "Allowed Number of Field History FIelds" will show as the "of X Selected". For example, if 40 is entered and 5 fields are selected, the label will say "(5 of 40 selected)". By default, Salesforce allows 20 fields to be selected. However, this limit can be increased by purchasing more.
